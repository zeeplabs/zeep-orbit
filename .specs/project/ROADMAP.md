# Roadmap

## M1 — MVP Core (current focus)

**Goal:** Schema YAML → REST API running on your PostgreSQL. CLI-driven. Docker Compose deploy.

Features:
- [ ] Config loader (apps.yaml)
- [ ] Schema parser + validation
- [ ] PostgreSQL schema provisioning (CREATE/ALTER TABLE)
- [ ] Auto REST CRUD per table (GET, POST, PUT, DELETE, GET/:id)
- [ ] JWT auth middleware per app
- [ ] HTTP reverse routing by app name
- [ ] CLI: `zeep apply`, `zeep list`, `zeep status`
- [ ] Docker Compose deploy (zeep-core + PostgreSQL)
- [ ] Basic query params: `?limit=`, `?offset=`, `?{field}=eq.{value}`

**Done when:** A frontend generated by an AI tool can define a schema, run `zeep apply`, and make authenticated REST calls against its own schema on company PostgreSQL — all in under 10 minutes.

---

## M2 — Developer Experience

- [ ] TypeScript SDK (`@zeep/client`)
- [ ] Official prompt snippets for Claude Code / Cursor / Lovable
- [ ] MCP server for zeep-core operations
- [ ] Web dashboard (app catalog, schema viewer, logs)
- [ ] Helm chart / Kubernetes deploy
- [ ] Schema migrations (alter existing tables safely)
- [ ] Filtering: `?{field}=gt.{value}`, `in.`, `like.`
- [ ] Sorting: `?order={field}.asc`

---

## M3 — Governance & Security

- [ ] Audit log (who did what, when, on which app)
- [ ] Role-based access per app (admin, editor, viewer)
- [ ] Corporate SSO integration (Google Workspace, Microsoft Entra ID)
- [ ] Rate limiting per app
- [ ] Schema change approval workflow

---

## M4 — Storage & Events

- [ ] File storage per app (S3/MinIO compatible)
- [ ] Signed upload/download URLs
- [ ] Webhook support (on insert/update/delete)
- [ ] Event bus integration

---

## Deferred

- GraphQL auto-generation
- Realtime subscriptions
- Edge functions
- Multi-region support
- Marketplace of app templates
